Friend Stalker API documentation

Authentication is simulated; in a real world application it would be a token header but here it's done by adding the x-user header containing the "authenticated" user's ID, e.g. to perform operations on behalf of the user foo you would include the x-user: foo header with your requests.

Any of the API calls below may produce a HTTP 500 text/plain response should an unexpected exception occur, e.g. a database server failure. The response body will hold the error description, if applicable.

Retrieve a list of users who have accepted the authenticated user's subscription requests

Response

HTTP statusFormatDescriptionBody
200application/jsonSuccessAn array of user objects. Each user object has the non-null keys "id" and "lastUpdated", the latter mapping to a millisecond-precision UNIX timestamp.
